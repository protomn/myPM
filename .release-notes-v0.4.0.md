> myPM v0.4.0 is the release where the loop closes for real: recall is pushed into every session instead of hoped for, knowledge genuinely compounds across repositories, the first-session traps are gone, and ‚Äî in the project's own spirit ‚Äî every claim below is measured, not assumed. The repo now runs myPM on itself; this release was partly designed by its own audit lessons, which are now nodes in the committed graph.

## Highlights

**Recall is pushed, not pulled.** `mypm init` wires a `SessionStart` hook that injects `mypm orient` ‚Äî the project description plus the most load-bearing, freshest decisions, lessons, patterns, and preferences in ~1k tokens ‚Äî into every Claude Code session before the first prompt. A memory you must remember to consult is not yet a memory.

**Knowledge compounds across repositories.** Point `MYPM_GLOBAL_ROOT` at a shared knowledge repo and its global-scope nodes (patterns, preferences) join every local `retrieve`, `orient`, and `search`. Local nodes win id collisions; other repos' project scopes never leak. A lesson learned in one repo, promoted to a global pattern, is recalled in the next repo on day one ‚Äî `git pull` is the sync protocol.

**The root is discovered, like `.git`.** Every command walks up from wherever you run it (`--root` > `$MYPM_ROOT` > discovery). Running from `repo/src/deep/` finds `repo/knowledge/`; running outside any myPM repo errors with the remedy. Read commands never create anything ‚Äî the silent-empty-graph-from-a-subdirectory failure mode is gone.

**`mypm doctor`.** Every hook in myPM is silent by design; silence is a feature in a hook and a bug in a diagnostic. Doctor checks root resolution, index freshness, the build pass, the git hook, the Claude Code hooks ‚Äî *including whether their executables actually resolve* (the classic venv failure) ‚Äî the AI/semantic extras, and the global root. Every failure prints its fix.

**Both loops are measured.** Every retrieve logs its bundle; `mypm feedback good|bad` rates the last one; the Stop hook detects when a later session actually *cites* a recalled node. `mypm stats` reports the write path (time-to-approve, enriched vs bare drafts) and the read path (Recall Win Rate + citation rate).

**A human-grade read surface.** `mypm show <id>` (fields, body, edges, lineage), `mypm search <terms>`, `retrieve --format text` ‚Äî no more `cat`-and-`grep` archaeology.

## First-session traps, fixed

Every one of these was reproduced against v0.3 before being fixed, and is now covered by tests:

- `capture --project` auto-links to the project node, so capture ‚Üí reflect ‚Üí distill promotes end-to-end (previously: guaranteed Gate-2 dead end on the very first use)
- `review approve --link` and `supersede --link` work (previously accepted, silently dropped)
- Decisions type from plain sentences ‚Äî `"we will use SQS instead of Kafka because ops burden"` ‚Üí `choice` + `rationale`; preferences type too
- Slug collisions get a content-hashed suffix instead of a false "already exists" quarantine; identical re-captures are still held
- Every blocked gate prints a copy-pasteable `fix:` command
- List fields (`alternatives="kafka; rabbitmq"`) coerce identically in capture and review ‚Äî found by dogfooding
- `.claude/settings.json` is **generated** with the installing interpreter pinned; a bare `mypm` in a hook command fails silently under Claude Code's PATH, forever, on every venv install

## Review, upgraded

- Interactive review shows each draft's **content and provenance** ‚Äî fields, body, source ‚Äî not just a title. An authorship gate where the author cannot see the text is a rubber stamp.
- `--type` / `--source` / `--project` filter the queue; `mypm review approve --all` bulk-promotes everything already passing Gate 2 (the verb for a backlog `/enrich-drafts` has finished with).

## Also in this release

- `bootstrap --enrich` now rescues rule-dropped prefilter survivors (feature/release-style subjects that carry decisions without choice verbs); empty free-pass runs explain themselves and quote the escape hatch with a call-count estimate
- `validate --errors-only` (CI mode) and `--all`; near-duplicate detection now uses inverted-index candidate pairing, capped per node
- Ready-made GitHub Actions workflow: `mypm/templates/ci/knowledge-validate.yml` ‚Äî knowledge errors fail the PR, judgment-call warnings stay advisory
- `mypm council` is now marked **EXPERIMENTAL** (sequential API calls; the doctrines as plain Claude Code subagents remain the supported path)
- The myPM repo itself now carries a live, committed `knowledge/` graph ‚Äî including the audit lesson that motivated root discovery, wired to the decision via a `motivates` edge

## Behavioral changes

- `--root` no longer defaults to `./knowledge`: it is discovered. Scripts relying on implicit root creation in cwd will now error with a remedy ‚Äî run `mypm init` once, or pass `--root` / set `MYPM_ROOT`.
- Only `mypm init` creates a root; all other commands require one to exist.
- `mypm/templates/settings.json` was removed; the file is generated per-machine at init. If yours predates v0.4 (bare `mypm` commands), regenerate: `rm .claude/settings.json && mypm init` ‚Äî `mypm doctor` will tell you.
- `validate` output is grouped and capped; CI scripts should pin `--errors-only`.
